# WebForge v0.2 重构实现计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将 WebForge 重构为符合 `learn-claude-code` 思路的 agent-first harness，重建 `.webforge` 契约，让 runtime 成为唯一主流程，并保持 `Codex + Claude Code` 双兼容与有边界的 `superpowers` 集成。

**Architecture:** 先重建 workspace schema 与核心类型，再把 session、deliverable、mailbox、planning、runtime 这些持久化与执行能力抽回 `src/core/`。CLI 只保留参数解析和结果渲染，`Agent` 收缩为执行门面，文档与实现同步对齐到同一套仓库契约。

**Tech Stack:** Node.js 18+, TypeScript, Commander, fs-extra, YAML, Vitest

---

**Spec Reference:** `docs/superpowers/specs/2026-03-30-webforge-v0.2-harness-design.md`

**Repository Note:** 当前目录没有 `.git/`。如果要按计划中的 checkpoint commit 执行，请先初始化 git；否则将每个 “Commit” 步骤视为可选检查点。
